「いつかやるかもしれない」候補リスト。実装したくなった時に上から検討する。各項目には背景・概算工数・判断ポイントを残す。

## 🎵 音声品質の向上（VOICEVOX または OpenAI TTS）

**現状**: Web Speech API (iPad の Kyoko Enhanced)。実用上はOK。

**選択肢**:

| 方法 | 工数 | コスト | 音質 | 備考 |
|------|------|-------|------|------|
| VOICEVOX で事前生成 mp3 | 1時間 | 完全無料 | ⭐⭐⭐⭐ | アプリ起動が必要、推奨話者「九州そら」「青山龍星」 |
| OpenAI TTS で事前生成 | 30分 | 約2円 | ⭐⭐⭐⭐⭐ | APIキー必要、`tts-1` / nova-shimmer 等 |
| Google Cloud TTS Neural2 | 1時間 | 無料枠内 | ⭐⭐⭐⭐ | アカウント設定が面倒 |

**実装イメージ**:
```
1. scripts/generate-audio.js を書く（cards.json を読み TTSに渡す）
2. audio/[category]/[id].mp3 を生成
3. app.js の speak() を mp3 再生に変更（フォールバックで Web Speech 維持）
4. sw.js の SHELL_ASSETS / 動的キャッシュに audio/ を追加
5. CACHE_NAME を v2 に上げる
```

**やるタイミング**: STさんから「機械音声が患者の理解を妨げる」フィードバックが出たら。

## 📝 機能追加

### 結果記録（匿名集計のみ）
- セッション終了時にカード別の正答情報を残せる UI
- ただし患者情報は**絶対に保存しない**方針堅持
- localStorage に集計用カウンタを置く程度なら可
- 「今日は何問やったか」程度のサマリ画面

### カスタム単語追加機能
- ST が独自の語を追加できる UI
- 画像はカメラから撮影・アルバムから選択
- localStorage に保存、JSON エクスポート可

### 動詞・形容詞カードの追加
- 仕様書 §9 にも記載
- 現状は名詞のみ
- 動詞は絵での表現が難しいので動画／GIF を検討する余地

### 文単位の呼称（短文音読）
- 「私はリンゴが好きです」のような文を提示
- 仕様書 §9 で言及済み
- TTSの自然さがより重要になる（音声品質向上が前提）

### 難易度モード切替
- 既存の親密度フィルタを「やさしい / 普通 / 難しい」の単純ボタンに簡略化
- 院内研修中のSTでも使いやすく

## 🔧 技術的改善

### Service Worker の更新通知
- 新バージョンがデプロイされた時、ユーザーに「更新があります」と通知して reload を促す UI
- 現状: 次回起動時に自動更新（タイムラグあり）
- 工数: 30分程度

### 画像の WebP 対応
- WebP は PNG より 30-50% 軽い
- iOS 14+ で対応
- 工数: 画像変換 + `<picture>` タグでフォールバック対応で 30分
- 効果: 7.8MB → 4-5MB 程度

### 多言語対応（英語UI）
- 海外研修・外国人ST対応
- カードは日本語のままで、UI のみ英語版
- 工数: 1-2時間

### CSS の整理（CSS Variables 拡張）
- 現状: `:root` に基本色のみ
- フォントサイズ・スペーシングも変数化すると Tier 3 デザイン作業が楽

## 🚀 配信・運用

### 独自ドメイン取得
- 現状: `kotoba-card.pages.dev`
- 例: `kotoba-card.jp` や `st-naming.app` 等
- Cloudflare Pages の Custom domain 設定で接続可能
- 年1000円程度

### 院内サーバーで運用
- Cloudflare 不要にしたい場合（情報セキュリティポリシー等）
- nginx で静的配信するだけで動く
- HTTPS は院内CA証明書が必要

## ❌ 検討して見送ったもの

| 項目 | 見送り理由 |
|------|-----------|
| React / Vue / Svelte 化 | 規模に対して過剰、ビルド工程が増える |
| TypeScript 化 | 同上、`'use strict'` で十分 |
| ネイティブアプリ化（.ipa） | Apple Developer 年$99、Mac 必要、過剰投資 |
| Firebase Hosting | Cloudflare Pages で十分、移行メリット無し |
| Cloudflare Workers でサーバーサイド処理 | 静的サイトで完結する設計 |
| 患者情報の保存・統計 | プライバシー・院内ポリシー |
| ゲーム的演出（スコアランキング） | 患者層に不適切 |
| カスタム音声録音（ST自身の声） | 工数大、Web Audio API + サーバー必要 |
